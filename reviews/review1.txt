+1. child1 - от подобных классов с номерами и child лучше избавиться, они не несут никакой смысловой нагрузки, ни чем не лучше, обращения по div c nth-child. Лучше, когда из названия класса понятно, что это за элемент. header, например, может быть размечен так.

<header class="header">
  <img class="header__logo"></div>
  <h1 class="header__title">Be recruited <span>Play college sports</span></h1>
  <div class="header__buttons">
  </div>
</header>

По классу сразу понятно, что это за элемент, а то, что у каждого элемента внутри header в названии класса в начале стотит header, дает также понять, где находится элемент. Для некоторых элементов это важно, логотип, например, может находиться в футере, шапке и еще где-то и иметь разную верстку.

+2. Текст в шапке - заголовок h1.

+3. section-header - нет смысла оборачивать в div.

+4. container-flex в блоке ATHLETS. Тут лучше сделать цифру через псевдоэлемент для h3. Она не является контентом, это, скорее, декоративный элемент.

+5. В header, чтобы текст нормально отображался на десктопе при изменении ширины, для текст надо ограничить ширину с помощью max-width, задать отступы по бокам небольшие, чтобы текст не прилипал к экрану. Вот https://codepen.io/sborjes/pen/xmPVVw

+6. Лишний div для изображения в блоке. Сразу за child1 пусть следует picture.

7. Текст в ATHLETS на 961. Для .section1 .child задаем padding-right: 15px. .section1 .text - max-width: 36em. Для section2 .child     padding-right: 280px.

8. .section3 .number-text - padding сверху в px, снизу тоже надо задать padding, на случай, если текста будет слишком много (он хотя бы не прилипнет к нижнему изображению.) Для блоков с изображением слева задаем для .section3 .number-text padding-right: 15px, для .section3 .text max-width: 630px. Там, где текст справа для .section3 .card2 .number-text padding-left: 15px, для .section3 .card2 .heading и а.section3 card2 .text max-width: 630px; margin-left: auto. На 961 получается все равно объективно получается говно, но в данных условиях лучше не сделаешь, поэтому можно оставить так. Либо написать для этого блока еще один медиа запрос на 1440, например, чтобы уменьшить отступы и размеры заголовка.

9. display: flex; flex-direction: column; не нужны для того, чтобы блоки шли вертикально. Это обычное поведение блочных элементов.

+10. Фоновым изображеням в шапке и футере задаем background-size: cover, чтобы фон всегда заполнял весь блок.

+11. .header .child1 - не нужен flexbox. Блочные элементы центрируются с помощью margin-left: auto; margin-right: auto. Посмотрите в других местах.
